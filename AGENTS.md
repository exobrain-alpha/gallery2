## App Description

## Technology Stack

- 客户端框架：Tauri 2.0
- Tauri 前端语言：React + TS
- 数据库：SQLite3 + rusqlite

## Project Structure

- 实现功能前，必须查阅对应官方文档，确认接口、模块与配置符合当前版本标准。
- 应用级控制、窗口控制、外部接口、认证授权、持久化、安全和系统集成能力必须由后端或平台层统一管理。
- 优先沿用现有模块边界和入口；新增模块、目录或入口前必须确认现有结构无法承载。
- 验证必须覆盖真实应用路径；局部验证必须说明范围和剩余风险。
- 采用非官方推荐方案或偏离既有结构时，必须说明原因、影响和替代方案。
- TypeScript 与 Rust 文件必须遵守职责单一原则；多功能或多模块大型文件必须拆分。

## Dependency Policy

- 默认不引入第三方依赖。如需引入第三方库，必须先说明用途、替代方案、包体积影响和维护风险，并与用户确认。
- 禁止为少量工具函数引入大型工具库。
- 禁止重复引入功能相近的库。
- 禁止引入未维护、低 star、来源不明或许可证不清晰的依赖。

## React Component Policy

- 组件名使用 PascalCase。
- 文件名与默认导出组件名保持一致。
- 组件要遵守职责单一原则。大型多功能单一组件，必须拆分。
- 前端组件禁止直接调用系统能力。

## TypeScript Policy

- 必须使用 TypeScript。
- 禁止使用 `any`，除非有明确注释说明原因。
- 公共组件 props 必须定义明确类型。
- 禁止在多个文件重复定义相同类型。
- 类型定义优先靠近使用位置；跨模块共享类型放入 `types/`。

## Icon Policy

- 操作图标统一使用 `lucide-react` 按需导入。
- 样式与图标资源必须按实际使用项逐一引入；禁止整包、整目录或通配导入。
- 应用中的图标最终必须以内嵌 SVG 形式渲染。

## Git Rules

- 分支命名：
  - `feat/xxx`
  - `fix/xxx`
  - `refactor/xxx`
  - `docs/xxx`
  - `chore/xxx`
- Commit 信息使用中文。
- Commit 格式：
  - `feat: 新增下载页`
  - `fix: 修复移动端导航样式`
  - `refactor: 重构首页区块组件`
  - `docs: 更新部署说明`
  - `chore: 调整构建配置`
- PR 合并使用 squash merge。
- PR 合并后删除本地和远程分支。
- 禁止将多个无关改动放入同一个 PR。
